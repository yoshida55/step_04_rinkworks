# 命名規則まとめ

## 基本ルール（会社規約）

- 区切りはアンダースコア（`_`）
- ハイフン不可
- タグ名を入れない
- ローマ字不可
- 極端な省略不可

---

## クラス名の形式

```
[何の]_[役割]
```

具体的に、何のセクションの何かがわかるようにする。

---

## 役割の接尾語一覧

| 役割                     | 接尾語         | 例                         |
| ------------------------ | -------------- | -------------------------- |
| section タグ             | `_area`        | `recipe_image_area`        |
| flex の親                | `_list`        | `recipe_image_list`        |
| list の中（単体）        | `_item`        | `sns_item`                 |
| list の中（入れ子 flex） | `_card`        | `recipe_card`              |
| タイトル                 | `_title`       | `recipe_diary_title`       |
| 説明文                   | `_description` | `recipe_diary_description` |
| 画像                     | `_image`       | `recipe_image`             |
| ボタン                   | `_button`      | `recipe_list_button`       |
| リンク                   | `_link`        | `footer_sns_link`          |
| 著作権                   | `_copyright`   | `footer_copyright`         |
| 注釈                     | `_note`        | `price_note`               |
| キャプション             | `_caption`     | `image_caption`            |
| SP 専用改行              | `sp_br`        | `sp_br`                    |
| ページ全体               | `wrapper`      | `wrapper`                  |
| ヘッダー                 | `header`       | `header`                   |
| フッター                 | `footer`       | `footer`                   |
| ナビ                     | `nav`          | `nav`                      |

---

## 「\_text」は使わない

「text」は曖昧になりやすいので、具体的な名前をつける。

```
❌ intro_text    → 何のテキスト？
⭕ intro_description  → 説明文
⭕ intro_note         → 注釈
⭕ intro_caption      → キャプション
```

「\_text」を使いそうになったら「これ本当は何？」と考える。

---

## ナビゲーションの構造

```html
<nav class="nav">
  <ul class="nav_list">
    <li><a class="nav_link">HOME</a></li>
    <li><a class="nav_link">ABOUT</a></li>
    <li><a class="nav_link">CONTACT</a></li>
  </ul>
</nav>
```

- `nav` → nav タグ自体
- `nav_list` → flex の親
- `nav_link` → 中のリンク

---

## flex の入れ子：\_item と \_card の使い分け

**1 段階の flex（中身が単体）**

```html
<div class="sns_list">
  <a class="sns_item">Instagram</a>
  <a class="sns_item">Twitter</a>
  <a class="sns_item">Facebook</a>
</div>
```

**2 段階の flex（中がまた flex になる）**

```html
<div class="recipe_list">
  <div class="recipe_card">
    <img class="recipe_card_image" />
    <p class="recipe_card_title"></p>
  </div>

  <div class="recipe_card">
    <img class="recipe_card_image" />
    <p class="recipe_card_title"></p>
  </div>
</div>
```

| 構造             | 名前                     |
| ---------------- | ------------------------ |
| 1 段階           | `_list` → `_item`        |
| 2 段階（入れ子） | `_list` → `_card` → 中身 |

## コメントルール

### セクション・ヘッダー・フッターには必ずコメントを入れる

後から見たとき「ここは何のブロックか」が一目でわかるようにする。

**形式：**

```html
<!-- ===============================================
 ここにセクションの説明を書く
=============================================== -->
```

**実際の例：**

```html
<!-- ===============================================
 ヘッダー：ロゴとナビゲーション
=============================================== -->
<header class="header">...</header>

<!-- ===============================================
 メインビジュアル：トップの大きな画像
=============================================== -->
<section class="main_visual_area">...</section>

<!-- ===============================================
 Recipe Diary紹介：サイトタイトルと説明文
=============================================== -->
<section class="recipe_diary_area">...</section>

<!-- ===============================================
 レシピ画像一覧：料理写真を横並びで表示
=============================================== -->
<section class="recipe_image_area">...</section>

<!-- ===============================================
 レシピ一覧ボタン：一覧ページへの導線
=============================================== -->
<section class="recipe_list_area">...</section>

<!-- ===============================================
 フッター：SNSリンクとコピーライト
=============================================== -->
<footer class="footer">...</footer>
```

**コメントに書く内容：**

- そのブロックの名前（何か）
- 補足説明（何を表示するか）

**例：** `レシピ画像一覧：料理写真を横並びで表示`

## 判断フローチャート

```
クラス名を付けたい要素がある
    │
    ├─ sectionタグ？ → [内容]_area
    │
    ├─ flexで子要素を並べる親？ → [内容]_list
    │
    ├─ 並んでる中の1つ1つ？ → [内容]_image, [内容]_link など
    │
    └─ その他の要素？ → [どこの]_[役割]
        │
        ├─ 見出し → _title
        ├─ 説明文 → _description
        ├─ 画像 → _image
        ├─ ボタン → _button
        ├─ リンク → _link
        └─ 著作権 → _copyright
```
